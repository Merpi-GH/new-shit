# Endpointهای مورد نیاز برای تکمیل پروژه طبق PDF

این موارد در فرانت‌اند ساخته شده ولی به API واقعی وصل نیستند.
لطفاً با تیم بک‌اند چک کنید که آیا این endpointها وجود دارند:

## 1. همکاری در فروش (بازاریابی) - PDF صفحه 16-17
- **نیاز:** لیست کالاهایی که فروشنده گزینه «فروش از طریق بازاریاب» را فعال کرده
- **احتمالی:** `GET /api/products/affiliate-bank/` یا مشابه
- فیلدهای مورد نیاز: نام کالا، عکس، قیمت، درصد کمیسیون بازاریاب، لینک اشتراک‌گذاری

## 2. ثبت آگهی - PDF صفحه 16
- **نیاز:** هنگام ثبت کالا اگر تیک «ثبت آگهی» فعال باشد، کالا در بخش آگهی‌ها هم نمایش داده شود
- **احتمالی:** فیلد `is_ad: true` در `POST /api/products/create/` + صفحه لیست آگهی‌ها `GET /api/ads/`

## 3. سیستم نوبت‌دهی (مشاغل خدماتی) - PDF صفحه 11
- **نیاز:** برای کالا/خدمت نوع «خدمات»، مشتری بتواند زمان نوبت انتخاب کند
- **احتمالی:** `GET /api/services/{id}/available-slots/` و `POST /api/reservations/`
- توضیح: model و bloc پایه (`ReservationBloc`, `ServiceBloc`) از قبل در پروژه ساخته شده ولی به API وصل نیست

## 4. اتصال به درگاه پرداخت شخصی - PDF صفحه 13
- **نیاز:** کاربر کد درگاه شخصی خود را وارد کند و ادمین آن را تأیید کند
- **احتمالی:** `POST /api/markets/{id}/connect-gateway/` با body شامل نوع درگاه (شخصی/آسود) و کد درگاه

## 5. انتشار/عدم انتشار/غیرفعال‌سازی فروشگاه - PDF صفحه 8-9
- **وضعیت فعلی:** به `queueMarket()` و `inactiveMarket()` وصل شده
- **نیاز به تأیید:** آیا این دو endpoint دقیقاً معادل «درخواست انتشار» و «عدم انتشار/غیرفعال» در PDF هستند؟
  - دکمه «انتشار» → باید دو گزینه «منتشر شود» / «عدم انتشار» بدهد
  - دکمه «غیرفعال» → باید جدا از «عدم انتشار» باشد (PDF این دو را متفاوت تعریف کرده)

## 6. راهنمای صوتی - PDF صفحه 3، 10، 14
- **نیاز:** فایل‌های صوتی راهنما برای هر صفحه + پلیر
- **احتمالی:** `GET /api/help-audio/{page_key}/` که لینک فایل صوتی برمی‌گرداند

## 7. پشتیبانی (چت با ادمین) - PDF صفحه 3
- **نیاز:** چت مستقیم کاربر با پشتیبانی آسود
- این می‌تواند از همان سیستم پیامرسان داخلی استفاده کند یا یک کانال جدا باشد - نیاز به مشخص شدن

---
**یادداشت برای فرانت‌اند:** هر کدام از این موارد که endpoint واقعی پیدا شد،
فقط لازم است در فایل مربوطه (مثلاً `marketer_api_service.dart`) تابع API
جایگزین شود؛ UI از قبل آماده است.
